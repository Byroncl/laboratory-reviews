# I18n Module 🌍

Handles application translations for English and Spanish.

## Overview

Provides endpoints to fetch translations used by the frontend.

## Architecture

⚠️ **Two parallel i18n systems** exist (issue to fix):

1. **TranslationService** (Singleton) — Used by guards/filters
2. **I18nService** (Request-scoped) — Used by modules

See [I18n Inconsistency Issue](../issues/i18n-inconsistency.md)

## Language Detection

Language is detected from `Accept-Language` header:

```
Accept-Language: es-MX,es;q=0.9,en;q=0.8
→ 'es'

Accept-Language: en-US,en;q=0.9
→ 'en'
```

## Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/i18n/translations/:language` | GET | Get translations for language |

## Frontend Usage (Angular)

```typescript
// Fetch translations
this.http.get('/i18n/translations/es').subscribe(translations => {
  // Use translations
  console.log(translations.auth.required);  // 'Autenticación requerida'
});
```
